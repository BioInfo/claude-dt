# Contributing to dt

Thanks for your interest in contributing to dt! Here's how to get started.

## Development Setup

```bash
git clone https://github.com/BioInfo/claude-dt.git
cd claude-dt
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

### Frontend (optional)

The web dashboard requires Node.js 18+:

```bash
cd web
npm install
npm run dev      # Dev server on :5173 (proxies /api to :8042)
npm run build    # Build to src/dt/static/
```

## Running Tests

```bash
pytest                    # Run all tests
pytest -x                 # Stop on first failure
pytest --cov=dt           # With coverage
```

## Project Structure

```
src/dt/
  cli.py          # Click CLI entry point
  db.py           # DuckDB schema and connection management
  ingest.py       # JSONL session parser and ingestion pipeline
  analyzers.py    # All analyzers, scoring, and recommendations
  config.py       # Paths and configuration
  api.py          # FastAPI backend for web dashboard
  static/         # Built React app (generated by npm run build)
web/              # React + TypeScript + Vite frontend source
tests/            # pytest test suite
```

## Guidelines

- Keep the flat module structure. No subpackages under `src/dt/`.
- All SQL uses DuckDB syntax. Use `_interval(days)` helper for date filtering.
- DuckDB does not support `?` placeholders in INTERVAL expressions; use f-strings with validated ints.
- Frontend uses Tailwind CSS (CDN), Lucide icons, and Recharts.
- Run `pytest` before submitting a PR. All 194+ tests must pass.

## Adding a New Analyzer

1. Add the analysis function to `src/dt/analyzers.py`
2. Add a CLI command to `src/dt/cli.py`
3. Add an API endpoint to `src/dt/api.py`
4. Add tests to `tests/`
5. Update the README

## Code Style

- Python: standard library conventions, type hints where helpful
- TypeScript: strict mode, functional React components
- Keep dependencies minimal

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Run tests (`pytest`)
5. Commit with a descriptive message
6. Push and open a pull request

## Reporting Issues

Open an issue on GitHub with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your Python version and OS
